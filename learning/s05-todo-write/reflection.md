# s05 Reflection

我的感受是，s05 的 TodoWrite 是一个简单的 Plan 模式内嵌。它确实会让 Agent Loop 变得更清晰，因为计划也被表达成一个普通工具，和 Bash、读写文件等工具走同一套 `tool_use` / `tool_result` 协议。

这种方式也方便后续变得更严谨：计划不再是 loop 外面的特殊逻辑，而是可以被模型调用、被测试观察、被上下文继续引用的结构化行为。

不过它现在还只是初步版本，所以还有很多值得思考的地方：什么时候应该强制 plan，plan 是否需要用户确认，plan 和执行权限应该如何联动，计划变更应该如何展示，失败后怎么回滚或重排，长期任务又应该如何保存。这些都不是 s05 当前要解决的问题，但它们说明 TodoWrite 是后续更完整 Plan / Task 能力的入口。
