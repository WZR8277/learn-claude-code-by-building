# s02 Tool Use — 个人反思

这一章没有带来特别强烈的感受，主要是让我初步建立了两个认识：文件工具需要先做调用路径校验；多个工具可以通过统一的 handler 映射完成分发，而不必在 Agent Loop 中分别处理每一种工具。

目前的实现仍然只是教学所需的最小工具系统。真实的工具管理通常还需要更完善的执行失败管理、沙箱隔离、权限问询，以及工具名称、描述和参数定义的持续优化。这些能力不应该提前塞进 s02，而应留到后续对应章节再逐步理解。

另外，工具应该返回多少内容并不只是工具自身的问题：返回过长会迅速占用模型上下文，截取过短又可能丢失完成任务所需的信息，因此工具结果长度最终也属于上下文管理的一部分。工具真正执行之前，通常还需要对请求字段做校验，并在合理范围内补充缺省值或给出清晰错误。s02 让我开始看到这些边界，但当前章节只保留教程中的最小实现，不提前加入这些后续能力。
