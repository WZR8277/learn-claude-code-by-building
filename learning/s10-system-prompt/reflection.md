# s10 Reflection

我的感受是，本次修改的“动态组装系统提示词”在目前项目进度里，动态这一点其实几乎没有真正发挥作用。因为当前会变的 system prompt 输入很少：除非模型主动写入记忆索引，或者运行中修改技能文件、工具集合等，否则同一个 agent loop 里的 system prompt 基本不会变化。

但从真实 Agent 系统来看，这个方向是合理的。对话推进过程中，工具权限准入导致工具集合变化、用户要求切换工作路径、模式切换等少数情况，确实可能让 system prompt 需要重新组装。不过我也认为，“每轮都更新 system prompt”这件事很有待商榷。很多动态变化的东西根本不适合进入 system prompt，它们更适合放在 tool result、当前 user turn 的附加上下文、程序内部状态或按需加载内容里。

所以这章我更关注的不是“当前 demo 里 prompt 是否真的频繁变化”，而是它把 prompt 从硬编码字符串改成了分段、可选择、可缓存的结构。这个结构本身为后续扩展提供了位置，但也需要警惕不要滥用。

另外我注意到，本次修改仍然尽量保持 prompt section 的固定顺序和固定前缀。为了 KV Cache 友好，真实系统里应该尽量保证 system prompt 前缀稳定，把易变内容放在后面或放到更合适的上下文位置。动态组装不等于频繁改写高优先级提示词。
