# s06 Reflection

我的感受是，Subagent 本质上就是“把 Agent Loop 包装成一个工具”。从父 Agent 的视角看，它不需要知道子 Agent 的全部执行过程，只需要拿到一个可关联的工具结果。

这个设计是合理的：普通工具补充的是本地文件、Shell 或其他外部信息，而 `task` 工具补充的是另一段模型驱动的探索结果。它仍然符合 `tool_use -> tool_result` 这条协议，只是 tool handler 内部会再调用大模型并运行一个独立循环。

教学版里的实现已经能看到几个关键边界：简单的轮次限制、子工具绑定、独立系统提示词、只返回摘要。但现实中的细节一定远比现在复杂，比如上下文要继承多少、权限如何冒泡给父 Agent 或用户、取消和超时如何联动、资源和费用怎么控制、是否允许并行或异步执行，以及递归委派应该如何限制。

所以我现在对 s06 的理解是：它先用最小实现讲清楚“父 Agent 把一段探索委托给独立上下文，并只接收结论”这个核心抽象；至于更复杂的工程问题，会在后续的上下文、后台任务、团队、任务系统和 worktree 等章节里继续展开。
