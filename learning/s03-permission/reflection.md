# s03 Permission — 个人观点

目前这些权限规则看起来都比较显然和简单：检查固定的危险字符串，命中后拒绝或询问用户。它们很适合展示“工具执行前必须经过权限判断”这条主线，但离生产级策略还有明显距离。

生产环境往往需要更复杂的分层判断，例如确定性的硬规则、对命令和路径的结构化分析、结合用户原始意图和当前环境的上下文判断，以及更细致的授权来源和优先级。对于规则难以判断的复杂文案或操作，也可以让较小、较快的模型做一次风险 double check。

不过，小模型更适合作为语义风险分类器或第二意见，而不是唯一的安全边界。模型仍可能误判或受到提示注入影响，因此高风险操作最终还需要硬拒绝规则、用户审批和系统级隔离共同约束。

本章的核心收获不是这几条关键词本身，而是先建立一条清晰的权限管线：明显危险的直接拒绝，需要判断的交给用户，其余调用才进入真实工具执行。
