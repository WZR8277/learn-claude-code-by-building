# s03：Permission

## 本章目标

在 s02 的工具分发前增加权限判断，让每个工具调用在执行前得到三种结果之一：硬拒绝、询问用户、直接允许。

## 相对 s02 的精确增量

- 新增硬拒绝列表与 `check_deny_list()`。
- 新增需要询问用户的 `PERMISSION_RULES` 与 `check_rules()`。
- 新增 `ask_user()` 和三关串联入口 `check_permission()`。
- Agent Loop 在 handler 调用前检查权限；拒绝时返回关联原调用 ID 的 `Permission denied.`。
- s02 中 Bash 自己的危险字符串判断移到权限层，工具实现恢复为只负责执行。

## 阅读 diff 时关注

1. 为什么权限判断必须位于 handler 之前。
2. 硬拒绝为何不会弹出审批，而规则命中才会询问。
3. 拒绝后为何仍要向模型返回一个 `tool_result`。
4. 为什么一次审批只控制当前工具调用。
5. 教学版字符串规则与操作系统沙箱之间的边界。

## 本章范围

严格跟随上游 s03：不增加规则配置文件、会话授权缓存、shell 语法解析、权限继承、审批 UI、Hooks 或真正的系统沙箱。
