# s11 Reflection

我的理解是，本次功能是在 Agent Loop 里嵌入错误处理，把模型调用作为一个可运行对象传入重试方法，从而可以在不重复执行工具副作用的前提下重复调用模型。

可重试的一般是 429 限流、529 服务器过载这类靠等待可能恢复的瞬态异常。这里的重试时间加入了指数退避，并且对 529 额外记录连续失败次数，达到阈值后可以切换到降级模型。

对于输入超长和输出超长，处理方式是不一样的。输入超长说明请求本身太大，所以会尝试一次更重的 reactive compact；但因为这已经是补救压缩，所以只重试一次，避免陷入空转。输出超长则说明模型已经返回了中断前的回答，第一次会先扩大输出预算重新请求；如果还是截断，才保留截断内容并提示模型从中断处继续，而且可以多接续几次。

初步看，这一章已经具备了一些错误恢复机制，但我感觉还不完善。比如错误类型是通过字符串匹配写死的，虽然设置了重试次数，但没有考虑用户对等待时间的容忍度，也没有更细的错误分类、取消机制或恢复策略选择。它更像是一个教学版的恢复骨架，而不是完整生产级错误处理。
