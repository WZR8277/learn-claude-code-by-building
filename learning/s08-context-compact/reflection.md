# s08 Reflection

我的感受是，上下文管理往往是渐进式的。本次实现了一些比较粗略、但是机制比较多的上下文压缩：有按消息数量裁剪，有旧工具结果微压缩，有大输出落盘，也有用模型做摘要式压缩。

不过这里面的参数还很值得继续思考，比如压缩比例、最大消息数、保留最近多少条工具结果这些数字，目前都还是经验值。现实系统里可能需要根据任务类型、模型上下文窗口、当前错误状态或者工具输出规模动态调整。

另外，本章也用到了模型去压缩历史。我相信在真实系统里，摘要模型的选择、摘要输出结构、需要保留哪些字段、如何验证摘要没有丢掉关键事实，都会是很重要的工程问题。

所以 s08 的重点不是某一个压缩函数，而是让我看到上下文管理本身是一套渐进式预算系统：先做便宜的裁剪和替换，必要时再让模型参与摘要，并且要始终保护工具协议和当前任务继续执行所需的信息。
